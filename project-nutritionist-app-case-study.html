Background

Problem Identified

Why this mattered

Professional nutritionists need rapid assessment tools to evaluate food, cosmetic, and household products for their clients. Manual composition research is time-consuming, often incomplete, and prone to inconsistency. The solution required automated product identification, accessible product data, and standardized quality scoring based on nutritionist expertise.

Constraints and risks

  • Product database coverage across four distinct categories with varying data availability.
  • ML model must be explainable and interpretable for professional clinical use.
  • On-device scoring for offline reliability and data privacy in clinical settings.
  • Cross-platform support (iOS and Android) with native performance expectations.
Our Solution

Integrated Scanning & Scoring System

Product strategy

We designed a seamless workflow: barcode scanning → product identification → composition analysis → ML-based quality scoring → color-coded risk visualization. Nutritionists receive validated product sheets with explainable scoring indicators, enabling fast clinical decision-making and client recommendations.

Multi-category supportBarcode recognitionComposition mappingML scoringRisk visualization

What we delivered

  • Flutter cross-platform mobile app for iOS and Android with native barcode scanning.
  • Spring Boot backend API with MySQL database storing product catalogs and nutritionist-validated datasets.
  • ML scoring engine built with Scikit-learn: Random Forest regression for quality score prediction, classification for status categorization.
  • JWT-based authentication and REST APIs for secure client-server communication.
Technical System

How we solved it technically

Mobile & Scanning

Flutter provides unified codebase for iOS/Android with native capabilities. Barcode scanning library integrates camera module, decodes product identifiers, and initiates server lookup. Responsive UI renders product sheets with color-coded risk indicators based on scoring output.

FlutterDartNative BridgeCamera API

ML Scoring Engine

Scikit-learn Random Forest models trained on nutritionist-validated product dataset. Regression pipeline predicts numerical quality scores; classification pipeline assigns status bands (Excellent/Good/Average/Poor). All model features engineered from product composition data with explainability through feature importance analysis.

Scikit-learnRandom ForestFeature EngineeringModel Calibration

Backend & Data

Spring Boot REST APIs handle product lookups, scoring requests, and user management. MySQL stores product catalogs across 4 categories with complete composition and dimension data. JWT tokens secure API endpoints. Product scoring system documents all dimensions, intervals, and status bands for clinical transparency.

Spring BootMySQLJWT AuthREST APIs
System Flow

Pipeline from scan to assessment

Step 01

User scans barcode from food, cosmetic, household, or supplement product.

Step 02

Mobile app sends identifier to backend for product lookup and composition retrieval.

Step 03

ML model extracts features from composition and generates quality score prediction.

Step 04

Classification model assigns status rating and constructs risk profile with color codes.

Step 05

App displays validated product sheet with all dimensions, scores, and clinical recommendations.
