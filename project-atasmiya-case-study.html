ATASMIYA
Halal Traceability Case Study

Cross-border halal traceability platform spanning suppliers, restaurant managers, and consumers. The project combined mobile and web products with anti-fraud controls and certification lifecycle management.

Background + Problem

Traceability gaps in certification flows

Industry issue

Certification verification was fragmented and difficult to trust across stakeholders. Restaurants and consumers needed fast proof of authenticity, while administrators required strict monthly control over batches and certificates.

Constraints

  • Multi-actor permissions and data ownership.
  • Mobile usage in variable network conditions.
  • Need for anti-fraud location validation.
  • Auditable flow from batch to consumer scan.
Technical Solution

System architecture and controls

Admin Back-Office

Next.js + Spring Boot admin plane for suppliers, lots, restaurants, and certificate allocation with operational visibility.

Mobile Workflows

Flutter app with manager and consumer modes: batch scan + verification via QR and secure profile-scoped operations.

Security + Integrity

JWT auth, AES-256 encryption, offline-first states, and GPS distance validation to detect suspicious claim attempts.

Deliverables + Impact

What was delivered and achieved

Deliverables

  • Complete admin control center.
  • Manager mobile certification flow.
  • Consumer QR verification journey.
  • Secure API and database backbone.

Impact

The platform improved trust, reduced manual verification overhead, and introduced consistent traceability evidence from supply chain input to consumer-facing confirmation.
