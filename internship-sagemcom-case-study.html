AI Mobile Internship

Background + Problem

Operational issue identified

Context

The R&D inventory process depended on manual product input and barcode steps. In fast-paced lab handling, this created repetitive data-entry friction and avoidable mismatches between physical items and records.

Main pain points

  • Manual typing produced frequent field inconsistencies.
  • Item lookup was slow during daily stock updates.
  • Image references were disconnected from product records.
  • No intelligent automation in existing mobile workflow.
Our Solution

Android + EfficientNet pipeline

Approach

I implemented an Android app with full CRUD, filtering, and local persistence, then integrated a TensorFlow Lite EfficientNet model to recognize products from camera input and prefill item metadata.

Technical execution

  • Camera stream preprocessing for stable inference.
  • EfficientNet TFLite model integration on device.
  • SQLite/Room-backed local inventory states.
  • Fallback manual override for low-confidence predictions.
  • Testing strategy with JUnit + Mockito for unit coverage, Room/integration tests for repository-database consistency, and Espresso UI tests for critical user journeys.
Deliverables + Impact

What was delivered and why it mattered

Deliverables

  • Production-ready Android inventory app.
  • AI-assisted product recognition flow.
  • Advanced search/filter and product CRUD lifecycle.
  • Local-first data layer for reliability.
  • Automated quality pipeline with JUnit, Mockito, Room integration tests, and Espresso UI scenarios.

Impact

The system significantly reduced operator friction, cut input errors, and improved inventory update speed from roughly 30 seconds to under 5 seconds for recognized products.
